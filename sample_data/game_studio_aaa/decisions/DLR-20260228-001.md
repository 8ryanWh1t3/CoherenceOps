# DLR-20260228-001: Patch Tuesday SLA Definition

| **Owner**     | Release Engineering (LA)                   |

| **Domain**    | Patch Cadence / Operations                 |

## Context

AetherForge currently has no formal SLA for critical hotfix deployment. Average time from sev-1 drift signal to deployed hotfix is 4.6 days, with a worst case of 11 days during the Season 3 "infinite ammo" exploit. This latency erodes player trust, triggers sponsor penalty clauses, and allows exploits to damage competitive integrity. Eclipse Dominion's live-service model demands a binding internal SLA.

## Options Considered

1. **48-hour SLA for sev-1 and high severity (selected)** -- from drift signal detection to deployed hotfix, 48h max. Applies globally.
2. **24-hour SLA** -- aggressive but requires 24/7 on-call across all studios. Annual staffing cost: +$1.8M.
3. **72-hour SLA** -- more realistic for current staffing but does not meet sponsor contract obligations (48h response clause).

## Decision

Adopt Option 1. Commit to 48-hour maximum from critical drift signal to deployed hotfix for sev-1 and high severity issues. SLA clock starts when the drift signal is classified by the on-call triage lead. All five studios participate in a follow-the-sun on-call rotation.

## Trade-offs

- (+) Meets sponsor contractual response requirements.
- (+) Reduces average exploit exposure window from 4.6 days to <2 days.
- (-) Requires follow-the-sun on-call; impacts Warsaw and Berlin teams on weekends.
- (-) Pressure to ship fast may increase regression risk.

## Assumptions

- **ASM-0003**: On-call staffing is sufficient across all time zones (minimum 2 engineers per studio per rotation).
- **ASM-0011**: Unified build pipeline (DLR-20260224-001) is operational, enabling any studio to produce a hotfix build.
- **ASM-0015**: Self-hosted runners can handle emergency hotfix builds with <15 min queue time.

## Kill Switch / Rollback

Suspend SLA with `WAIVER` label applied to the specific incident. WAIVER requires VP-level approval and triggers a post-incident review within 72h. SLA suspension does not affect non-critical patch cadence.
