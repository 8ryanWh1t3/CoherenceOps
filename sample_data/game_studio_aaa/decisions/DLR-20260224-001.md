## Context

AetherForge Interactive operates three independent CI/CD pipelines across LA, Montreal, and Tokyo studios. Integration failures at merge points account for 23% of all build breaks, costing an estimated 1,400 engineering-hours per quarter. Eclipse Dominion's weekly content cadence demands a unified artifact pipeline that can produce a single shippable build from any branch within 45 minutes.

## Options Considered

1. **Unified pipeline on GitHub Actions (selected)** -- single workflow definition, region-aware runners, shared artifact cache.
2. **Hub-and-spoke model** -- LA as primary, Montreal/Tokyo push deltas. Lower migration cost but single point of failure.
3. **Status quo with improved merge gates** -- add pre-merge validation without changing tooling. Cheapest but does not address root cause.

## Decision

Adopt Option 1. Migrate all three studios to a unified GitHub Actions pipeline with self-hosted runner fleets in each region. Shared artifact cache via S3 cross-region replication. Target: single green build within 45 min across all studios by 2026-04-15.

## Trade-offs

- (+) Eliminates integration-merge class of build failures (~23% of total).
- (+) Single source of truth for build definitions.
- (-) Migration window of 6 weeks; parallel pipeline operation increases cloud spend by ~$140K.
- (-) Tokyo team loses custom Perforce bridge; requires P4-to-Git LFS migration for large assets.

## Assumptions

- **ASM-0011**: All three studios can converge on a single branching strategy within Q1 2026.
- **ASM-0015**: Self-hosted runner capacity in Tokyo DC can sustain peak build load (>200 concurrent jobs).

## Kill Switch / Rollback

Revert to per-studio pipelines. Each studio retains legacy pipeline definitions in `legacy-ci/` branches for 90 days post-migration. Rollback time: <4 hours per studio.
