# DLR-20260309-003: Localization Divergence Correction Policy

## Context

The JP localization divergence discovered in DLR-20260309-002 exposed a systemic gap: AetherForge has no automated mechanism to detect when localized content diverges from the EN-canonical source. The current process relies on manual spot-checks during milestone reviews, which failed to catch 42 JP dialogue entries that had drifted over an entire season. With Eclipse Dominion supporting 14 languages and a weekly content cadence, manual review alone cannot scale. The CANON-v1.2 tag system exists but is not enforced by tooling.

## Options Considered

1. **Automated diff tool with 48-hour review policy (selected)** -- build an automated diff pipeline that flags divergences between EN-canonical source and all localized dialogue files within 48 hours of any content commit. Human reviewer required to approve or override.
2. **LLM-assisted translation verification** -- use large language model to compare semantic meaning across translations. More sophisticated but higher false-positive rate and ongoing API costs.
3. **Expanded manual review team** -- hire 4 additional localization QA reviewers. Reliable but does not scale with content velocity.
4. **Localization freeze windows** -- restrict narrative content changes to biweekly windows. Reduces drift risk but conflicts with weekly content cadence.

## Decision

Adopt Option 1. Build and deploy an automated diff tool ("CanonGuard") that:

1. Monitors all commits to dialogue files in the content repository.
2. Within 48 hours of any EN-canonical change, generates a structured diff against all 14 language packs.
3. Flags divergences exceeding a configurable threshold (initial: any non-whitespace change to dialogue or lore text).
4. Requires a localization reviewer to acknowledge or override each flag before the next content milestone ships.
5. Publishes weekly divergence reports to the Narrative and Localization leads.

All narrative content must be reviewed against EN-canonical source within 48 hours of localization completion. This is a standing policy, not a time-limited waiver.

## Trade-offs

- (+) Catches localization drift within 48 hours instead of months.
- (+) Scales with content velocity -- automated, not headcount-dependent.
- (+) Enforces CANON tag system with tooling rather than process alone.
- (+) Low ongoing cost after initial build.
- (-) $95K tooling investment.
- (-) May generate false positives for intentional localization adaptations (cultural localization).
- (-) Adds a mandatory 48-hour review window to the localization pipeline.

## Assumptions

- **ASM-0012**: Localization teams can deliver accurate translations within 48 hours for emergency corrections when given EN-canonical source and full context. This policy depends on that turnaround time being realistic at steady state, not just sprint conditions.

## Kill Switch / Rollback

Disable CanonGuard automation and fall back to manual review process (pre-existing milestone spot-checks). The tool can be disabled per-language or globally. Rollback time: immediate (feature flag).
