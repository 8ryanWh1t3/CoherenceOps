# DLR-20260302-001: Tournament Infrastructure Scaling Plan

| Field         | Value                                      |
|---------------|--------------------------------------------|
| **Status**    | Accepted                                   |
| **Date**      | 2026-03-02                                 |
| **Owner**     | Infrastructure Engineering (LA)            |
| **Studios**   | LA (lead), Tokyo (APAC infra)              |
| **Domain**    | Esports Infrastructure                     |

## Context

The Eclipse Dominion Championship Series Week 10 (EDCS W10) finals are scheduled for 2026-03-14. Expected peak concurrent viewership: 800K+ (up from 520K at EDCS W6). Standard infrastructure supports 600K concurrent with auto-scaling. Tournament requires 3x normal capacity provisioned 48h in advance to guarantee zero-downtime during broadcast. OrionTel is title sponsor with $2.4M at stake in the broadcast window.

## Options Considered

1. **Pre-provision 3x normal capacity across 3 regions (selected)** -- dedicated tournament fleet in us-east-1, ap-northeast-1, eu-central-1. Budget: $1.8M.
2. **Rely on auto-scaling policy (DLR-20260224-002)** -- trust the scaling policy to handle the spike. Zero additional cost but untested at 800K concurrent.
3. **2x provision + auto-scale buffer** -- compromise approach. Lower cost ($1.1M) but thinner margin for error.

## Decision

Adopt Option 1. Pre-provision 3x normal capacity 48 hours before EDCS W10. Dedicated fleet isolated from public matchmaking (see DLR-20260302-002). LA manages us-east-1 and eu-central-1 fleets; Tokyo manages ap-northeast-1. War room staffed 2h before broadcast through 1h post-event.

## Trade-offs

- (+) Guarantees capacity headroom even at 150% of projected peak.
- (+) Isolation from public matchmaking prevents tournament issues from affecting casual players.
- (-) $1.8M budget for a single event weekend.
- (-) Over-provisioning if actual peak is below projections.

## Assumptions

- **ASM-0005**: Peak concurrent users will not exceed 2.5M (800K tournament + 1.7M public).
- **ASM-0002**: Average session length remains ~42 min (affects connection pool sizing for tournament viewers).
- **ASM-0013**: AWS reserved capacity is available in all 3 target regions at 48h notice.

## Success Metrics

| Metric                         | Target       | Baseline (W6) |
|--------------------------------|--------------|---------------|
| Tournament server uptime       | 100%         | 99.7%         |
| Peak concurrent supported      | 800K+        | 520K          |
| Player-perceived latency (p99) | <60ms        | 78ms          |
| Broadcast stream interruptions | 0            | 2             |

## Kill Switch / Rollback

Scale-down to 2x capacity if utilization remains below 40% after 2 hours of broadcast. Full decommission of tournament fleet within 6 hours post-event. Cost saved on early scale-down: ~$400K.
