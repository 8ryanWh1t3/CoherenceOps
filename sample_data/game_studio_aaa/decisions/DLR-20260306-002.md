# DLR-20260306-002: Emergency Hotfix Bypass Protocol

| Field         | Value                                      |
|---------------|--------------------------------------------|
| **Status**    | Accepted (WAIVER)                          |
| **Date**      | 2026-03-06                                 |
| **Owner**     | Engineering Governance (LA)                |
| **Studios**   | LA, Montreal, Tokyo, Warsaw                |
| **Domain**    | Patch Cadence / Release Engineering        |

> **WAIVER**: Code review bypass for critical live incidents.
> **WAIVER-EXPIRY**: 2026-06-06 (90 days). Must be re-ratified or expires to standard process.

## Context

Eclipse Dominion's live service cadence requires rapid response to severity-1 incidents affecting player experience. During W10-W11, three sev-1 issues required hotfixes but were delayed an average of 18 hours waiting for full QA regression suites (4.5h per run, 2 runs required). The existing patch cadence policy (DLR-20260228-001) mandates full regression before any production deployment. This rigidity cost an estimated 42,000 player-sessions and contributed to the SLA breach tracked in DRIFT-20260313-001.

## Options Considered

1. **Bypass protocol with dual senior engineer approval (selected)** -- sev-1 hotfixes skip full QA regression if 2 senior engineers (Staff+ level) approve the change set. Targeted smoke tests still required.
2. **Parallel QA with speculative deployment** -- deploy hotfix to canary while regression runs. Rollback if regression fails. Higher risk of bad deploy reaching players.
3. **Dedicated fast-track QA lane** -- separate regression suite (1h) for hotfix-class changes. Higher upfront investment.
4. **Status quo** -- keep full regression requirement. Accept 18h+ hotfix latency.

## Decision

Adopt Option 1 as a time-limited waiver. Sev-1 hotfixes may bypass full QA regression under these conditions: (a) two Staff+ engineers sign off on the diff, (b) targeted smoke tests covering the affected subsystem pass, (c) the hotfix is deployed to 5% canary for 15 minutes before full rollout, (d) full regression is run post-deployment within 6 hours. All bypasses are logged in the incident tracker with justification.

## Trade-offs

- (+) Reduces sev-1 hotfix deployment time from ~18h to ~2h.
- (+) Recovers an estimated 14,000 player-sessions per sev-1 incident.
- (-) Increased risk of shipping regressions in hotfix code.
- (-) Creates a precedent for process bypass that may be culturally difficult to retract.
- (-) Dual-approval requirement may bottleneck on Staff+ availability during off-hours.

## Assumptions

- **ASM-0003**: Existing QA regression suite catches >95% of gameplay-affecting bugs before production. This bypass accepts reduced coverage for speed.
- **ASM-0015**: Self-hosted runner capacity in Tokyo DC can sustain peak build load -- relevant because smoke tests must run on all regional runners simultaneously.

## Success Metrics

| Metric                                | Target        | Baseline       |
|---------------------------------------|---------------|----------------|
| Sev-1 hotfix deployment time (p95)    | <3 hours      | 18 hours       |
| Post-bypass regression failure rate   | <5%           | N/A (new)      |
| Bypasses per quarter                  | <6            | N/A (new)      |
| Player-sessions recovered per hotfix  | >10,000       | 0 (delayed)    |

## Kill Switch / Rollback

Revoke bypass protocol immediately. All hotfixes revert to full QA regression requirement. Any in-flight bypass deployments are halted at canary stage pending full regression. Activation: Engineering Governance lead or any VP-level decision-maker.
