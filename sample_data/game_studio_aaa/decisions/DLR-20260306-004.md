## Context

Following the Week 10 tournament latency incident, competitive community sentiment deteriorated sharply. The Sentiment Anomaly Index (SAI) reached 82 -- breaching the OrionTel renegotiation threshold of SAI>75 (clause 7.2). Community grievances cluster around three themes: (1) latency-affected match outcomes, (2) lack of transparency in server selection, (3) no mechanism for retroactive fairness correction. A coordinated protest by the Eclipse Dominion Competitive League (EDCL, ~48,000 members) called for a player boycott of the W11 tournament. OrionTel's brand manager has requested a written response plan within 72 hours.

## Options Considered

1. **Three-part transparency package (selected)** -- public latency dashboard, match replay system, retroactive MMR adjustment for affected W10 matches.
2. **Cash compensation** -- in-game currency grant to all W10 tournament participants. Fast but does not address structural issue.
3. **Tournament replay** -- re-run affected W10 brackets. Logistically complex; estimated 2-week delay to Season 4 schedule.
4. **Public apology + roadmap** -- acknowledge issue and commit to long-term fix. Low cost but insufficient for OrionTel and EDCL.

## Decision

Adopt Option 1. Implement a three-part competitive fairness package:

1. **Public Latency Dashboard** (ship by 2026-03-13) -- real-time per-match latency metrics visible to all players. Historical data for last 30 days.
2. **Match Replay System** (ship by 2026-03-27) -- full server-side replay for ranked and tournament matches. Accessible via web portal.
3. **Retroactive MMR Adjustment** (execute by 2026-03-10) -- identify all W10 tournament matches where server-side latency exceeded 150ms for >30% of match duration. Nullify those matches from MMR calculations. Notify affected players.

## Trade-offs

- (+) Directly addresses all three community grievance clusters.
- (+) Demonstrates proactive fairness commitment to OrionTel.
- (+) Match replay system has long-term value for esports ecosystem.
- (-) Total cost of $680K across all three deliverables.
- (-) Retroactive MMR adjustment may create secondary complaints from players who benefited from nullified matches.
- (-) Match replay system increases server-side storage by ~8TB/month.

## Assumptions

- **ASM-0002**: APAC player base latency tolerance remains at <=150ms for competitive play. Used as the threshold for MMR adjustment eligibility.
- **ASM-0009**: Current matchmaking algorithm distributes skill tiers evenly across server instances. Required for MMR nullification to be statistically fair.
