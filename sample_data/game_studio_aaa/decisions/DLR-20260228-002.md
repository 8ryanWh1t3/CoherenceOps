## Context

Eclipse Dominion's ranked mode reports a 6.2% cheater encounter rate (player-reported, validated at 4.1%). Current EasyAntiCheat v3 detects ~93% of known cheat signatures but struggles with kernel-level memory manipulation and custom aimbots. The competitive community has flagged anti-cheat as the #1 issue in the last two quarterly surveys. Retaining the 1.8M ranked player base ($38/yr ARPU) requires a material upgrade.

## Options Considered

1. **Upgrade to EasyAntiCheat v4 with kernel-level detection (selected)** -- adds ring-0 monitoring, behavioral heuristics, and hardware fingerprinting.
2. **Switch to Vanguard (Riot-style)** -- always-on kernel driver. Superior detection but significant player privacy backlash risk.
3. **Server-side statistical detection only** -- no client-side component. Zero privacy concerns but ~78% detection rate (insufficient).

## Decision

Adopt Option 1. Upgrade to EAC v4. Deploy kernel-level detection module as opt-in for ranked mode (required for ranked, optional for casual). Warsaw QA team leads regression testing across supported OS matrix (Win10, Win11, Steam Deck/Proton). Target: 99.5% detection rate.

## Trade-offs

- (+) Detection rate improvement from 93% to projected 99.5%.
- (+) Ranked-only kernel requirement limits privacy pushback.
- (-) Kernel module introduces compatibility risk with certain anti-virus software.
- (-) $800K implementation cost including license upgrade and QA.

## Assumptions

- **ASM-0009**: Player base accepts kernel-level anti-cheat for ranked mode (based on competitor precedent -- Valorant Vanguard acceptance rate >87%).

## Success Metrics

| Metric                          | Target       | Baseline      |

## Kill Switch / Rollback

Disable kernel module via client patch (pre-staged rollback build). Fall back to EAC v3 detection. Rollback deployment time: <6 hours globally. Kernel module can be disabled server-side for specific regions independently.
