# DLR-20260327-001: Narrative Review Gate for Live Events

| Field         | Value                                      |
|---------------|--------------------------------------------|
| **Status**    | Accepted                                   |
| **Date**      | 2026-03-27                                 |
| **Owner**     | Narrative Design (Montreal)                |
| **Studios**   | Montreal (lead), LA (live ops), Berlin (loc)|
| **Domain**    | Narrative Canon / Live Operations          |

## Context

The Season 3 Rift Collapse and subsequent localization divergence (DRIFT-20260320-001) revealed that live event content could ship without any narrative team review. The Rift Collapse event was authored by the LA Live Ops team using placeholder lore text that was never reconciled with Montreal's canonical narrative bible. When JP localization adapted the placeholder text, they introduced three canon contradictions that went viral. The Lore Consistency Framework (DLR-20260320-001) addresses the localization pipeline, but does not gate the upstream problem: live event content creation itself operates outside narrative governance. Season 4 plans include 8 live events over 12 weeks, each with narrative-integrated content (dialogue, item descriptions, environmental storytelling). Without a review gate, every live event is a potential canon breach. CANON-v1.4 formalizes the updated narrative rules, but rules without enforcement are aspirational.

## Options Considered

1. **Mandatory narrative sign-off gate in live event pipeline (selected)** -- all live event content requires Narrative Design sign-off before entering localization or deployment. Gate implemented as a Jira workflow state with 24h SLA.
2. **Narrative embed in Live Ops team** -- station a narrative designer in the LA Live Ops team full-time. Good coverage but single point of failure; does not scale to multi-studio events.
3. **Post-event narrative audit** -- review live event content after deployment and patch contradictions retroactively. Low friction but damage occurs before correction.
4. **AI-assisted canon checking** -- build an NLP tool that checks content against the narrative bible. Promising future state but 4+ months to build; does not exist yet.

## Decision

Adopt Option 1. Implement a mandatory narrative sign-off gate in the live event content pipeline. Every piece of live event content (dialogue, item text, environmental narrative, quest descriptions, cinematic scripts) must receive explicit approval from Montreal Narrative Design before it enters the localization queue or deployment pipeline. Gate implemented as a Jira workflow state ("Narrative Review") with a 24h SLA for standard content and 4h SLA for emergency/hotfix content. Narrative Director has veto authority; escalation path to Creative Director for disputes.

## Trade-offs

- (+) Prevents canon contradictions at source, before localization amplifies them.
- (+) Formalizes narrative governance that was previously informal/optional.
- (+) 24h SLA is tight enough to not block the weekly content cadence.
- (-) Adds 24h to live event content pipeline (accepted as necessary quality cost).
- (-) Montreal Narrative Design becomes a bottleneck if understaffed during crunch.
- (-) Veto authority may create friction with Live Ops team accustomed to autonomy.

## Assumptions

- **CANON-v1.4**: Canon update narrative review gate and incident SOP -- this decision operationalizes CANON-v1.4's review requirements.
- **DLR-20260320-001**: Lore Consistency Framework provides the canonical source of truth that reviewers check against.

## Kill Switch / Rollback

Bypass narrative gate via Jira workflow override (requires Narrative Director + Creative Director dual approval). Override flag: `narrative-gate-bypass`. All bypassed content flagged for retroactive review within 48h. Rollback time: immediate (Jira workflow toggle).
