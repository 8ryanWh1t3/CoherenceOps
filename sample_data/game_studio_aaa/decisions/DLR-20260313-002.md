## Context

The pay-to-win allegations (DLR-20260311-002) and Week 10 competitive fairness protests (DRIFT-20260305-001) have created urgent demand for a comprehensive statistical audit of Eclipse Dominion's ranked mode. While the independent third-party audit (Digital Fairness Labs) focuses on premium item properties, AetherForge needs its own internal data-driven analysis of ranked mode health across all dimensions: loadout win rates, MMR distribution fairness, input device parity (controller vs. M+KB), and regional performance variance. The Season 4 weapon balance pass (DLR-20260224-004) introduced 14 new loadout configurations that have not been statistically validated at scale.

## Options Considered

1. **Montreal-led 2-week comprehensive audit (selected)** -- dedicated analytics sprint covering win rates, MMR, input device parity, and regional variance. Montreal's gameplay analytics team leads with LA data engineering support.
2. **Automated anomaly detection dashboard** -- build real-time dashboards that flag balance outliers. Better long-term but 6-week build time does not address immediate need.
3. **Community-sourced data analysis** -- partner with trusted community analysts. Novel approach but data access and methodology concerns.
4. **Defer to DFL audit results** -- wait for the independent audit to inform internal analysis. Saves cost but delays actionable insights by 4+ weeks.

## Decision

Adopt Option 1. Montreal gameplay analytics team executes a 2-week comprehensive audit sprint (2026-03-14 to 2026-03-28) covering:

1. **Loadout Win Rates** -- per-loadout win rate analysis across all 147 loadout configurations in ranked play. Flag any loadout with >52% overall win rate or >54% at any MMR bracket.
2. **MMR Distribution** -- validate that MMR distribution follows expected bell curve. Identify any compression or inflation artifacts from the W10 retroactive adjustment (DLR-20260306-004).
3. **Input Device Parity** -- compare win rates and KDA between controller and mouse+keyboard players at equivalent MMR tiers.
4. **Regional Performance** -- analyze whether region-specific latency patterns (DLR-20260306-003) create systematic win rate bias.

Results published internally by 2026-03-28; public summary by 2026-04-04.

## Trade-offs

- (+) Provides AetherForge-owned data to complement the DFL independent audit.
- (+) 2-week sprint delivers actionable insights on tight timeline.
- (+) Covers dimensions beyond the DFL scope (MMR, input device, region).
- (-) $160K cost (Montreal team diversion + LA data engineering support).
- (-) Montreal team diverted from Season 4 mid-season balance work (2-week slip risk).
- (-) Results may surface additional balance issues requiring remediation.

## Assumptions

- **ASM-0009**: Current matchmaking algorithm distributes skill tiers evenly across server instances. If this assumption is invalid, win rate analysis by loadout will be confounded by skill-tier sampling bias.

## Kill Switch / Rollback

The audit itself is a read-only analysis -- there is nothing to roll back. Any balance changes recommended by the audit will be implemented as separate DLRs with their own kill switches. If the audit reveals critical imbalances requiring emergency intervention, the interim fix process from DLR-20260311-002 applies.
