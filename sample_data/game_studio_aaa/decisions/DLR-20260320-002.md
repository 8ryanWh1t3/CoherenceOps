# DLR-20260320-002: Automated Regression Gate for Patches

| Field         | Value                                      |
|---------------|--------------------------------------------|
| **Status**    | Accepted                                   |
| **Date**      | 2026-03-20                                 |
| **Owner**     | QA Engineering (Warsaw)                    |
| **Studios**   | Warsaw (lead), LA (pipeline integration)   |
| **Domain**    | Quality Assurance / Release Engineering    |

## Context

Warsaw QA bottleneck (DRIFT-20260319-001) caused patch turnaround to balloon from the 48h SLA to 96h during W11-W12, directly breaching the Patch Tuesday SLA (DLR-20260228-001). Root cause: Warsaw's 14-person QA team manually executes 2,800 regression test cases per patch. During the monetization crisis, three emergency hotfixes plus two scheduled patches created a queue of 5 concurrent validation jobs. EVID-0005 (Warsaw QA Pipeline Throughput Analysis) confirmed that manual regression accounts for 68% of total QA cycle time. The Patch Cadence SLA Revision (DLR-20260313-001) tightened the target to 36h, making manual-only QA structurally impossible at current patch velocity.

## Options Considered

1. **Automated regression gate with tiered test suites (selected)** -- automate 70% of regression tests (smoke, integration, performance) via Selenium + custom game-client harness. Manual QA reserved for gameplay feel, visual fidelity, and edge-case scenarios.
2. **Double Warsaw QA headcount** -- hire 14 additional QA engineers. Addresses throughput but not cycle time; adds $980K/yr.
3. **Outsource regression to external QA vendor** -- contract Keywords Studios or Pole To Win for regression overflow. Fast ramp but quality control risk and IP exposure.
4. **Reduce regression scope** -- cut test suite to 1,200 critical-path cases only. Fastest fix but increases escape rate.

## Decision

Adopt Option 1. Build an automated regression gate that runs 1,960 of 2,800 test cases (70%) automatically on every patch submission. Automated suite divided into three tiers: Tier 1 (smoke, 400 tests, <15 min), Tier 2 (integration, 1,100 tests, <90 min), Tier 3 (performance, 460 tests, <45 min). A patch must pass all three tiers before entering manual QA for the remaining 840 cases. Warsaw QA team transitions from full-regression to exploratory and gameplay-feel testing. Target: patch turnaround <24h (below 36h SLA).

## Trade-offs

- (+) Reduces QA cycle time from 96h to projected <24h.
- (+) Frees Warsaw QA for higher-value exploratory testing.
- (+) Automated tests run 24/7; eliminates timezone dependency on Warsaw business hours.
- (-) $340K upfront for test automation framework and game-client harness.
- (-) 6-week build-out period; manual QA continues as-is during transition.
- (-) Automated tests cannot assess gameplay "feel" -- false negatives possible for balance issues.

## Assumptions

- **EVID-0005**: Warsaw QA throughput analysis accurately reflects current bottleneck distribution.
- **ASM-0003**: Patch Tuesday SLA 48h response window -- this decision targets 24h, exceeding the revised 36h SLA.

## Kill Switch / Rollback

Disable automated gate via CI pipeline flag (`qa-auto-gate: false`). All patches route to full manual regression at Warsaw. Rollback time: <5 minutes (pipeline config change). Warsaw retains full manual test suite documentation.
