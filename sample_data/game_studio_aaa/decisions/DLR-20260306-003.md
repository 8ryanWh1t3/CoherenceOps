# DLR-20260306-003: Cross-Region Matchmaking Rebalance

| Field         | Value                                      |
|---------------|--------------------------------------------|
| **Status**    | Accepted                                   |
| **Date**      | 2026-03-06                                 |
| **Owner**     | Multiplayer Systems (Tokyo + LA)           |
| **Studios**   | Tokyo, LA                                  |
| **Domain**    | Netcode / Matchmaking                      |

## Context

Eclipse Dominion's global matchmaking pool routes players to the lowest-latency available server regardless of region. Since Season 4 launch, APAC player complaints about cross-region latency have increased 340% (W8-W11). Root cause: NA/EU server saturation during peak hours forces APAC players onto cross-region instances with 180-220ms latency -- well above the 150ms competitive threshold (ASM-0002). APAC DAU has declined 4.2% over W9-W11, concentrated in JP and KR markets. Tokyo studio flagged this in DRIFT-20260304-001.

## Options Considered

1. **Region-preferred matchmaking with 90s fallback (selected)** -- prefer same-region servers; only route cross-region when queue time exceeds 90 seconds.
2. **Hard region lock** -- players only match within their region. Eliminates latency complaints but fragments population, increasing queue times for low-population regions.
3. **Dynamic latency ceiling** -- match cross-region only if predicted latency <120ms. Technically elegant but requires real-time latency prediction model (3-month build).
4. **Additional APAC server capacity** -- add 40% more APAC instances. Addresses symptom but not root algorithm issue.

## Decision

Adopt Option 1. Modify matchmaking to prefer same-region servers as primary pool. If no match is found within 90 seconds, expand to cross-region pool with a 180ms latency hard ceiling. Display estimated latency to players before match acceptance when cross-region. Deploy in two phases: Phase 1 (APAC-only, 2026-03-13), Phase 2 (global, 2026-03-27).

## Trade-offs

- (+) Reduces APAC latency complaints by an estimated 60%.
- (+) Preserves cross-region fallback for low-population windows.
- (+) Minimal revenue impact -- this is a quality-of-life improvement.
- (-) Queue times may increase 15-25 seconds during off-peak in smaller regions (OCE, SA).
- (-) Cross-region friend groups may experience split matchmaking pools.
- (-) Two-phase rollout requires parallel matchmaking configs for 2 weeks.

## Assumptions

- **ASM-0002**: APAC player base latency tolerance remains at <=150ms for competitive play. This decision directly protects this threshold.
