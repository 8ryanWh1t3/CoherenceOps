# CoherenceOps - Outer-Edge Boundaries

CoherenceOps is an operational layer for **Truth / Reasoning / Memory** in organizational decision flow.

It is explicitly **NOT**:
- A runtime AI safety monitor
- A model control plane
- A security product
- A cryptographic attestation system for runtime actions
- A replacement for existing GRC tooling

CoherenceOps **IS**:
- Pre-decision / pre-merge governance via structured artifacts (DLR/RS/DS/MG)
- Drift -> Patch mechanics (versioned patches, no overwrite)
- Fast "why retrieval" and institutional memory reconstitution
- A measurable coherence index and adoption thresholds

Boundary rule:
> If it happens after execution, it's observability.
> If it prevents or structures execution, it's governance.
